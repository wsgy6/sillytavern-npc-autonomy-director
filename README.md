# NPC 多角色自主性导演

这是一个可直接作为 GitHub / GitLab 第三方仓库安装的 SillyTavern 扩展，用来同时控制多个 NPC 的长期驱动力、阶段演化、当前目标与“当前正在做什么”的行动状态。

## 这次升级重点

- 目标系统升级为“前期 / 中期 / 后期”三层目标链，并加入完成锁。
- 前期目标负责解决眼前小问题，完成后才会更新。
- 中期目标负责统领多个前期目标，只有形成实际改变后才会更新。
- 后期目标负责维持宏大愿景，默认几乎不变，除非完成重大阶段推进。
- 优先尝试调用 SillyTavern 的静默生成接口，根据角色个性、驱动力、当前状态生成目标；若不可用则自动回退到模板池。

- 从“单角色状态”升级为“多受控角色独立状态”。
- 可新增、指定、切换、删除受控角色。
- 每个角色拥有独立的：
  - 长期驱动力
  - 当前目标
  - 穿搭演化阶段
  - 行为演化阶段
  - 当前行动
  - 历史记录
- 每个角色的阶段文本都可以直接输入修改：
  - 穿搭四阶段
  - 行为四阶段
  - 早/中/后期目标池
- 每轮主 AI 回复都会被要求输出一个结构化行动区块，用于同步所有受控角色当前行动。
- 新增悬浮行动窗，可点击放大查看当前所有受控角色行动。

## 三层目标链与锁定规则

- 前期目标：面向当前局势中的具体小问题，例如试探、争取位置、确认信任对象。
- 中期目标：面向阶段性改变，例如让别人逐渐依赖自己、改变互动结构、为更强表态铺路。
- 后期目标：面向宏大愿景，例如稳定掌控局势、塑造鲜明存在感、建立长期关系结构。
- 三层目标都不会因为日常活动而直接改写，必须先判定“完成”，才允许进入下一次生成。
- 日常活动不加锁，但提示词会要求主 AI 主动通过行动推动前期目标完成，并让前期目标持续呼应中期与后期。

### 目标变化频率

- 前期目标：完成一个现实小问题后即可变化，变化最快。
- 中期目标：累计到实际改变后才变化，变化速度居中。
- 后期目标：只有宏大目标获得显著推进后才变化，几乎不变。

### 作用链路

- 后期目标定义角色长期想成为什么、掌握什么、改变什么。
- 中期目标把后期目标拆成一个阶段性方向，负责引导接下来一段时间的前期目标。
- 前期目标把中期目标落实成眼前可行动、可完成的小问题。
- 日常行动继续保持灵活，但要围绕前期目标发力，借由小胜利逐步释放 NPC 的自主性。

## 行动区块机制

扩展会向主提示词中注入固定要求，让主 AI 在每轮回复末尾追加以下结构：

```text
<npcad-actions>
角色名: 一句话行动描述
角色名: 一句话行动描述
</npcad-actions>
```

说明：当前版本同时兼容 `角色名: ...`、`角色名：...` 与 `角色名:: ...` 三种写法；推荐优先使用单冒号写法，最直观也最稳定。

如果开启“自动隐藏行动区块”，扩展会在消息渲染前把这段结构化区块从正文中移除，只保留到扩展状态与悬浮窗中展示。

## 使用方式

1. 打开扩展设置面板。
2. 在“角色管理”中新增一个或多个受控角色。
3. 选中某个角色后，编辑它的：
   - 目标
   - 驱动力
   - 当前行动
   - 穿搭/行为关键词
   - 穿搭四阶段
   - 行为四阶段
   - 早/中/后期目标池
4. 保持“要求主 AI 输出行动区块”为开启。
5. 对话推进后，悬浮窗会显示各受控角色的最新行动。
6. 点击悬浮窗或放大按钮，可查看全部角色当前行动明细。
7. 可在全局设置里调整悬浮窗宽度与最大高度。
8. 悬浮窗支持拖拽移动；非可交互聊天上下文（单聊/群聊）时会自动收缩，进入单聊或群聊后再展开查看当前行动。

## 仓库结构

仓库根目录直接包含：

- `manifest.json`
- `index.js`
- `style.css`
- `README.md`

## 安装方法

1. 在 GitHub 新建一个公开仓库。
2. 将本目录内所有文件上传到仓库根目录。
3. 确认仓库根目录直接显示：
   - `manifest.json`
   - `index.js`
   - `style.css`
4. 在 SillyTavern 的扩展安装入口填写该仓库地址。
5. 安装后刷新页面。

## 验证方式

### 1. 多角色验证

- 新增 2 个以上受控角色。
- 点击不同角色标签，确认面板中的三层目标链、阶段、行动内容会随当前角色切换。
- 删除某个角色后，确认其不再出现在列表与悬浮窗中。

### 2. 三层目标链验证

- 进入扩展面板后，确认总览区出现“当前短期目标”和“三层目标链”。
- 连续推进若干轮，确认前期目标会在完成后更新，中期/后期不会每轮乱跳。
- 若当前环境支持 SillyTavern 静默生成接口，目标应能体现角色个性、驱动力与当前状态；若接口不可用，仍应从模板池稳定生成。

### 3. 阶段可调验证

- 修改任意角色的“穿搭四阶段”或“行为四阶段”文本。
- 点击“同步提示词”。
- 继续对话数轮后，确认角色变化遵循新的阶段文本，而不是旧默认值。

### 4. 行动同步验证

- 保持“要求主 AI 输出行动区块”为开启。
- 生成一轮回复后，确认：
  - 扩展面板中“当前行动”被刷新。
  - 悬浮窗中显示了各角色当前行动。
  - 若开启“自动隐藏行动区块”，聊天正文中不应保留 `<npcad-actions>` 区块。

### 4.1 反馈闭环验证

- 让主 AI 输出带有明显动作关键词的行动块，例如：

```text
<npcad-actions>
角色A: 先观察对方反应，再试探性接近。
角色B: 悄悄改变穿衣风格，准备正式亮相。
</npcad-actions>
```

- 预期结果：
  - 包含“观察 / 试探 / 接近”等词的角色，行为进度会额外推进。
  - 包含“改变 / 风格 / 打扮”等词的角色，外观进度会额外推进。
  - 面板历史记录里会新增“行动反馈”条目。

### 4.2 目标推进与重评验证

- 让主 AI 输出带有“完成 / 达成 / 解决 / 突破”等词的行动描述，或多轮推进直到双进度都大于 `70`。
- 然后执行 `/npc-auto-status`，观察以下字段：
  - `当前目标层`
  - `目标是否完成`
  - `前期目标 / 中期目标 / 后期目标`
- 预期结果：
  - 命中完成信号时，当前活跃目标会被推进。
  - 当外观进度与行为进度都大于 `70` 时，会触发一次“目标重评”。
  - 历史记录里会新增“目标重评”条目。
  - 当前目标层会从 `early` 逐步推进到 `middle`、`late`，全部完成后显示 `completed`。

### 5. 悬浮窗验证

- 确认右下角出现“受控角色行动”悬浮窗。
- 点击放大按钮或悬浮窗内容，确认弹出完整行动面板。
- 点击角色卡后，确认当前编辑角色切换成功。

### 6. 指令验证

在聊天中输入：

- `/npc-auto-status`
- `/npc-auto-step`
- `/npc-auto-reset`

确认它们分别能输出状态、推进当前角色、重置当前角色。
